# Hermes 模型编排体系 v2.0 — 能力分工架构

> 954个模型 · 10个Provider · 6大能力层 · 4大业务场景
> 核心原则：**按能力分工，不按平台划分**

---

## 一、六大核心能力层

### 🧠 第一层：核心大脑（统筹规划、复杂推理、长程编程）

| 角色 | Provider | 模型 | 触发条件 |
|------|----------|------|---------|
| **主控大脑** | 智谱 GLM | glm-5-turbo / glm-5.1 | 默认所有任务入口 |
| **超长文本架构师** | DeepSeek | deepseek-v4-pro | 文本>50K token / 完整代码库分析 / 长剧本 |
| **多模态项目总管** | Kimi | kimi-k2.6 | 同时处理文本+代码+图像的复合任务 |

**路由规则**：
```
任务入口 → 判断文本量
├── < 8K tokens   → 智谱 GLM (默认)
├── 8K-100K tokens → DeepSeek V4-Pro
└── > 100K tokens  → Kimi K2.6 (128K) 或 DeepSeek (百万上下文)
```

---

### 📝 第二层：文本与内容工厂（剧情、文案、本地化）

| 角色 | Provider | 模型 | 触发条件 |
|------|----------|------|---------|
| **对话与剧情工厂** | 腾讯混元 | hunyuan-pro / hunyuan-turbo | NPC对话 / 分支剧情 / 任务文本 / 批量文案 |
| **全球语言专家** | OpenRouter | gpt-5 / claude-opus-4.7 / gemini-3-pro | 多语言翻译 / 本地化 / 文化合规审查 |

**路由规则**：
```
内容生成需求 → 判断语言和量级
├── 中文批量内容    → 腾讯混元 (高并发低成本)
├── 英文/日文/韩文  → OpenRouter (顶级模型)
└── 特定风格文案    → 对应语言的顶级模型
```

---

### 👁️ 第三层：视觉与多模态流（美术资产、图像理解、视频）

| 角色 | Provider | 模型 | 触发条件 |
|------|----------|------|---------|
| **视觉总监** | 阿里百炼 | qwen-image-2.0-pro / wanx2.7 | 角色/场景/UI生成 / 关键帧 / 美学评估 |
| **视觉试验场** | 硅基流动 | FLUX.1-Schnell / SDXL / LoRA模型 | 批量初稿 / 风格探索 / 低成本试错 |

**路由规则**：
```
视觉需求 → 判断阶段和质量要求
├── 最终成品（高质量）→ 阿里百炼 (万相2.7 / Qwen-VL)
├── 初稿/批量生产     → 硅基流动 (FLUX / SDXL)
├── 图像理解/评估     → 阿里百炼 (Qwen3-VL)
└── 视频生成          → 阿里百炼 (万相2.7) + ComfyUI
```

---

### 🔊 第四层：音频与语音工厂（配音、音效、音乐）

| 角色 | Provider | 模型 | 触发条件 |
|------|----------|------|---------|
| **配音导演** | MiniMax | MiniMax-M2.7 | TTS角色配音 / 旁白 / AI语音交互 |
| **音乐总监** | MiniMax | Music 2.6 | BGM / 环境音效 / UI交互音效 / 洗脑神曲 |

**路由规则**：
```
音频需求 → 判断类型
├── 人声/TTS        → MiniMax M2.7 (Speech)
├── 音乐/BGM        → MiniMax Music 2.6
├── 音效(SFX)       → Stable Audio (通过硅基流动或独立调用)
└── 语音克隆        → MiniMax VoiceClone
```
