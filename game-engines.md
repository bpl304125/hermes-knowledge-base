## 一、Godot Engine
| 项目 | 详情 |
|------|------|
| **仓库** | godotengine/godot |
| **Stars** | 110,545 |
| **语言** | C++ (核心), GDScript / C# / C++ (脚本) |
| **许可证** | MIT |
| **Forks** | 25,271 |
| **活跃度** | 极高（18k+ open issues / 极高频更新） |

### 核心特性
- 完全开源免费的2D/3D一体化引擎，无任何分成或授权费。
- **自研GDScript**：类Python语法的脚本语言，学习曲线极低；同时支持C#、C++（GDExtension）和可视化脚本。
- **专用2D引擎**：基于像素和2D坐标系的真正2D渲染器，而非3D降维模拟。
- **节点/场景系统**：一切皆节点，组合场景极为灵活，支持继承和实例化。
- **内置编辑器**：完整的IDE级编辑器，含动画编辑器、粒子编辑器、TileMap、Shader编辑器等。
- **多平台导出**：Windows、macOS、Linux、Android、iOS、Web、游戏主机（第三方支持）。
- Vulkan + OpenGL 渲染后端。

### 适用场景
- 独立游戏开发者和小型团队的首选。
- 2D游戏开发（公认最佳开源方案之一）。
- 快速原型和Game Jam。
- 需要完全掌控、不想被授权费锁定的商业项目。

### 学习曲线
- **低到中等**。GDScript极易上手，节点系统直观。3D高级特性需要一定学习投入。

## 二、Bevy
| 项目 | 详情 |
|------|------|
| **仓库** | bevyengine/bevy |
| **Stars** | 46,013 |
| **语言** | Rust |
| **许可证** | Apache-2.0 |
| **Forks** | 4,555 |
| **活跃度** | 极高（3.3k+ issues / 社区极活跃） |

### 核心特性
- **纯ECS架构**（Entity Component System）：所有游戏逻辑围绕System对Component的查询展开，数据驱动设计。
- **Rust语言**：内存安全、零成本抽象、无GC停顿，天然适合高性能场景。
- **模块化插件系统**：所有功能（渲染、音频、UI、物理）都是可插拔的Plugin。
- **支持2D和3D**，渲染基于wgpu（Vulkan/Metal/DX12/WebGPU）。
- **热重载**：开发时修改资源/代码即时生效。
- **编译期调度**：System依赖关系在编译期确定并行执行顺序。

### 适用场景
- Rust生态的游戏开发者。
- 追求极致性能和数据驱动架构的项目。
- 需要高度模块化/可定制引擎的技术型项目。
- 学习和实践ECS架构的优秀平台。

### 学习曲线
- **中等到高**。Rust本身有陡峭学习曲线，ECS范式与传统OOP差异大。文档和示例在快速完善中。引擎尚未稳定（v1.0未发布）。

## 三、Unreal Engine
| 项目 | 详情 |
|------|------|
| **仓库** | EpicGames/UnrealEngine (私有，需Epic账号) |
| **语言** | C++ (核心), Blueprints (可视化脚本) |
| **许可证** | 源码可用（Source Available），收入超100万美元需支付5%分成 |
| **社区** | 最大的游戏引擎社区之一 |

### 核心特性
- **业界AAA标准**：Nanite虚拟几何体、Lumen动态全局光照、MetaHuman角色系统。
- **Blueprints可视化脚本**：非程序员可完成完整游戏逻辑。
- **完整的工具链**：关卡编辑器、材质编辑器、动画蓝图、行为树、Niagara粒子系统等。
- **C++源码开放**：可深度定制引擎底层。
- **Epic Games生态**：Quixel Megascans免费库、Epic在线服务、免费示例项目。
- **多平台**：全平台支持（主机/PC/移动/VR）。

### 适用场景
- AAA级3D游戏开发。
- 影视级实时渲染（虚拟制片、建筑可视化）。
- 需要顶级画质的项目。
- 中大型团队（工具链集成度高）。

### 学习曲线
- **高**。功能极其庞大，C++接口复杂。Blueprints降低了入门门槛，但精通需要大量时间。

## 四、PlayCanvas
| 项目 | 详情 |
|------|------|
| **仓库** | playcanvas/engine |
| **Stars** | 15,664 |
| **语言** | JavaScript / TypeScript |
| **许可证** | MIT |
| **Forks** | 1,870 |
| **活跃度** | 高（610 issues / 持续迭代） |

### 核心特性
- **纯Web优先引擎**：基于WebGL 2.0 / WebGPU，无需插件在浏览器运行。
- **WebXR支持**：内置VR/AR体验支持。
- **glTF生态**：完整的glTF 2.0工作流，支持3D Gaussian Splatting。
- **云端编辑器**：基于浏览器的协作编辑环境（类似Google Docs式的游戏开发）。
- **物理引擎**：集成ammo.js (Bullet Physics WASM编译版)。
- **轻量级**：压缩后约1.5MB，加载极快。

### 适用场景
- 浏览器即玩的3D游戏（Web游戏/IO游戏）。
- Web端3D产品展示/配置器/可视化。
- VR/AR Web体验。
- 需要跨平台零安装分发的互动内容。

### 学习曲线
- **低到中等**。前端开发者上手极快；传统游戏开发者需适应Web工作流。

## 五、O3DE (Open 3D Engine)
| 项目 | 详情 |
|------|------|
| **仓库** | o3de/o3de |
| **Stars** | 9,090 |
| **语言** | C++ (核心), Lua / Python (脚本) |
| **许可证** | Apache-2.0 |
| **Forks** | 2,465 |
| **活跃度** | 中等偏高（3.5k+ issues） |

### 核心特性
- **AAA级开源引擎**：由Amazon Lumberyard捐赠给Linux基金会，Apache 2.0无商业限制。
- **多渲染器**：Atom Renderer（PBR/光线追踪），支持Vulkan/Metal/DX12。
- **Gem模块系统**：功能以Gem形式组装，按需启用。
- **Python脚本**：工具链和编辑器工作流支持Python。
- **网络层**：内置多人游戏网络框架。
- **程序化内容生成**：植被、地形等自动生成工具。

### 适用场景
- AAA级别3D项目（想用开源替代Unreal的团队）。
- 需要无分成、无授权费的商业3D游戏。
- 高保真模拟和可视化项目。
- 需要深度定制引擎的大型团队。

### 学习曲线
- **高**。功能复杂、文档和教程相对较少，社区较小，入门门槛类似Unreal但没有Blueprints。

## 六、LÖVE (Love2D)
| 项目 | 详情 |
|------|------|
| **仓库** | love2d/love |
| **Stars** | 8,285 |
| **语言** | C++ (引擎内核) / Lua (脚本) |
| **许可证** | Zlib (BSD式宽松) |
| **Forks** | 596 |
| **活跃度** | 稳定（仅117 open issues，成熟稳定） |

### 核心特性
- **纯2D游戏框架**（非编辑器引擎），通过Lua脚本编程。
- **极简设计**：无GUI编辑器，纯代码编写，API简洁优雅。
- **Lua脚本**：极简语法，易学，LuaJIT性能优异。
- **功能完备**：图形（硬件加速）、音频、物理（Box2D集成）、键盘/鼠标/手柄、网络等一应俱全。
- **单文件分发**：可将游戏打包成单一可执行文件。
- **成熟稳定**：自2008年开发至今，API极稳定。

### 适用场景
- 2D独立游戏（如《Baba Is You》《Move or Die》均使用Love2D开发）。
- 快速原型和Game Jam。
- Lua爱好者或想学习游戏开发的新手。
- 不需要复杂IDE的小型2D项目。

### 学习曲线
- **低**。Lua易学，框架API简洁。无GUI编辑器，纯代码开发方式让程序员非常舒适。成熟度高，文档和社区资源丰富。

## 七、Flax Engine
| 项目 | 详情 |
|------|------|
| **仓库** | FlaxEngine/FlaxEngine |
| **Stars** | 6,721 |
| **语言** | C++ (核心), C# (脚本) |
| **许可证** | 专有（源码可用，收入超过一定额度需分成） |
| **Forks** | 671 |
| **活跃度** | 中等（751 issues） |

### 核心特性
- **C++/C# 双语言**：高性能C++核心 + C#脚本（类似Unity的开发体验）。
- **高质量渲染**：PBR、全局光照、体积雾、后处理等特性齐全。
- **可视化脚本**：内置Visual Scripting，类似Blueprint。
- **跨平台**：Windows、Linux、macOS、Xbox、PlayStation、Switch、Android、iOS。
- **原生编辑器**：自研编辑器，启动快、界面流畅。
- **对Unity开发者友好**：API和编辑器布局与Unity高度相似，迁移成本低。

### 适用场景
- 寻找Unity替代品的C#开发者。
- 需要AAA画质但不想用Unreal的中型项目。
- 希望有商业支持且引擎源码可见的团队。

### 学习曲线
- **中等**。Unity开发者几乎零成本迁移。C#脚本式开发降低了C++的学习需求。

## 八、Panda3D
| 项目 | 详情 |
|------|------|
| **仓库** | panda3d/panda3d |
| **Stars** | 5,119 |
| **语言** | C++ (核心), Python / C++ (脚本) |
| **许可证** | BSD |
| **Forks** | 867 |
| **活跃度** | 中等（347 issues / 成熟引擎） |

### 核心特性
- **迪士尼+CMU出品**：历经验证的工业级引擎（用于《Toontown Online》等迪士尼MMO）。
- **Python优先**：Python作为主要脚本语言，极大降低入门门槛。
- **完整3D管线**：骨骼动画、粒子系统、Shader、碰撞检测、物理集成（Bullet）。
- **场景图架构**：灵活的节点树管理渲染和变换。
- **无编辑器**：纯代码驱动，适合程序员主导的开发流程。
- **跨平台**：Windows、macOS、Linux均支持。

### 适用场景
- Python开发者进入3D游戏开发的最佳入口。
- 学术/教育用途（计算机图形学教学）。
- 快速3D原型开发。
- 对编辑器依赖低、以编程为主的团队。

### 学习曲线
- **低到中等**。Python极友好，文档成熟，但无编辑器意味着需要更多编程工作。
