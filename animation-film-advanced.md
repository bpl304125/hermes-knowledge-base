## 一、Motion Matching 运动匹配

### 1.1 核心原理

传统动画状态机问题：状态爆炸(走/跑/转弯/加速/减速×方向×坡度)，手工过渡繁琐。
Motion Matching方案：抛弃状态机，在大量动画帧中实时搜索最佳匹配。

```
每一帧:
1. 提取当前角色状态特征向量
2. 在动画数据库中搜索最相似帧
3. 混合(inertialize)到目标帧
4. 播放目标帧及其后续帧
5. 下一帧重复
```

### 1.2 特征向量设计

```
特征向量(约20-30维):
├── 轨迹特征(最重要):
│   ├── 未来0.5s/1.0s/1.5s位置
│   └── 未来0.5s/1.0s/1.5s朝向
├── 脚部特征:
│   ├── 左脚位置/速度
│   └── 右脚位置/速度
└── 过去轨迹(提供动量信息)
```

### 1.3 搜索加速

```
暴力搜索 O(N): 每帧遍历百万帧动画 → 太慢
KD-Tree/球树: 特征空间最近邻搜索 → O(log N)
分层搜索: 粗搜索(每隔N帧)→细搜索→最优候选
PCA降维: 20维→8维主成分，搜索速度提升10倍
```

### 1.4 惯性混合 (Inertialization)

传统线性混合问题：两只脚同时滑动
```
Inertialization:
1. 源动画继续播放(不混合)
2. 目标动画从匹配帧开始
3. 计算源动画速度/加速度
4. 目标动画各关节添加虚拟弹簧(继承源速度)
5. 弹簧阻尼系数衰减，自然过渡
```

### 1.5 行业应用

| 游戏 | 方案 | 效果 |
|------|------|------|
| **The Last of Us Part II** | 自研MM | 业界标杆 |
| **For Honor** | 自研MM | 战斗系统丝滑 |
| **Paragon** | UE MM原型 | Epic开源 |
| **UE5.4+** | 内置MM | 插件化 |
| **Unity** | Kinematica | 实验阶段 |

## 二、程序化动画技术

### 2.1 IK 脚部放置

```
每帧脚步调整:
1. 从脚部骨骼向下射线检测地面
2. 计算IK目标 = 射线命中点 + 脚部偏移
3. 限制IK目标在最大步幅内
4. Two-Bone IK求解
5. 平滑移动(速度×dt)

防止滑动: 脚落地后锁定世界位置(Planting)
骨盆补偿: 双脚都会贴地时，骨盆高度 = min(左脚IK偏移, 右脚IK偏移)
```

### 2.2 物理动画混合

```
动画驱动 + 物理修正:

上半身: 纯动画驱动(精确控制)
下半身: 动画 + 布娃娃物理混合
受击部位: 完全物理(布娃娃)
         ↓ 恢复
动画姿态: 弹簧力拉向动画目标姿态

混合权重 = f(受力大小):
  轻推: 动画90% + 物理10%
  重击: 动画10% + 物理90%
  死亡: 完全物理(布娃娃)
```

### 2.3 注视系统 (Look At)

```
多层注视优先级:
1. 眼睛旋转(Damp Track约束, 最高优先级)
2. 头部旋转(权重减少，更大范围)
3. 脊柱旋转(大幅转向)
4. 身体转向(超过脊柱阈值)

目标选择:
- 附近其他角色/玩家
- 感兴趣的物体
- 威胁方向(警戒状态)
- 闲逛时注视点(随机采样)
```

### 2.4 程序化步态适配

```
地面法线 → 脚部倾斜匹配
坡度 → 身体前倾(上坡)/后仰(下坡)
窄道 → 手臂收拢
头顶障碍 → 自动蹲伏

Blend Space参数化:
横轴: 速度(-5~5m/s)  → 后退/静止/行走/慢跑/冲刺
纵轴: 转向角(-180~180°) → 左转/直行/右转
```

## 三、实时渲染高级技术

### 3.1 全局光照方案对比

| 方案 | 质量 | 性能 | 动态 | 适用 |
|------|------|------|------|------|
| **烘焙Lightmap** | 极高 | 极低(运行时) | ❌ | 静态场景 |
| **Light Probes** | 中 | 低 | ❌ | 动态角色GI |
| **SSGI(屏幕空间)** | 中 | 中 | ✅ | 辅助 |
| **Lumen(UE5)** | 高 | 高 | ✅ | 次世代 |
| **硬件光追RTXGI** | 极高 | 极高 | ✅ | 高端PC |
| **VXGI(体素)** | 中 | 中高 | ✅ | 中端 |

### 3.2 大气散射

```
Rayleigh散射(小粒子，如空气分子):
- 散射蓝光(短波)远多于红光(长波)
- → 天空是蓝色的原因
- 散射强度 ∝ 1/λ^4

Mie散射(大粒子，如气溶胶/水滴):
- 与波长无关(白光散射)
- → 云是白色的原因
- 在太阳周围产生光晕(Halo)

实现: Precomputed Atmospheric Scattering (Bruneton 2008)
```

### 3.3 体积云/雾

```
体积云: Ray Marching + Worley噪声
体积雾: 
├── 指数高度雾: density = e^(-height/H)
├── 体积光(God Rays): 屏幕空间射线行进
└── 局部体积雾: 球形/盒形，混合Worley噪声
```

## 四、电影摄影理论

### 4.1 虚拟摄像机参数

| 参数 | 含义 | 电影常用 | 游戏常用 |
|------|------|---------|---------|
| **焦距(mm)** | 视角宽度 | 35mm(标准)/85mm(人像)/24mm(广角) | 60-90 FOV |
| **光圈(f-stop)** | 景深 | f/2.8(浅景深)/f/11(深景深) | 可调 |
| **快门角度** | 运动模糊量 | 180°(标准电影模糊) | 可变 |
| **ISO** | 感光度/噪点 | 800(暗光)~100(亮光) | 模拟噪点 |

### 4.2 三点布光

```
Key Light(主光) ─── 最亮，定义主要阴影方向
Fill Light(辅光) ── 暗侧补光，减弱阴影
Rim Light(轮廓光)── 从背面分离角色与背景

光比: 主光:辅光 = 4:1(高反差/黑色电影) 到 1.5:1(柔和/喜剧)
```

### 4.3 Rembrandt 光

特点：脸颊出现倒三角光斑
设置：主光偏移45°，高于眼平30°

### 4.4 色彩理论

```
色轮关系:
├── 互补色: 红-青, 蓝-黄, 绿-品红(高对比，超级英雄)
├── 类似色: 蓝-青-绿(和谐，自然/平静)
├── 三色: 红-蓝-黄(均衡，动画常见)
└── 冷暖对比: 暖色前景+冷色背景(深度感)

Teal & Orange(蓝橙) — 好莱坞最常用色彩方案
暗部偏蓝(阴影)，亮部偏橙(肤色)，对比强烈
```

### 4.5 构图法则

| 法则 | 方法 | 效果 |
|------|------|------|
| **三分法** | 主体放在1/3交点 | 自然平衡 |
| **引导线** | 道路/栏杆指向主体 | 视觉引导 |
| **景深** | 浅景深突出主体 | 焦点明确 |
| **负空间** | 主体周围留白 | 强调主体 |
| **Dutch Angle** | 倾斜摄像机 | 不安/紧张 |
| **180°法则** | 对话双方不跨线 | 空间连贯 |

## 五、音效设计

### 5.1 Foley 拟音

用日常物品模拟电影音效：
- 掰芹菜 → 骨折
- 椰子壳敲击 → 马蹄声
- 揉搓玉米淀粉 → 雪地脚步
- 皮革摩擦 → 身体移动

### 5.2 音效分层 (近/中/远)

```
爆炸音效设计:
层1(近距): 冲击波(低频轰隆) — 0-10米
层2(中距): 碎片飞溅(中频碎裂) — 10-50米
层3(远距): 尾部回声(高频衰减) — 50-200米
层4(极远): 低沉的隆隆声(仅低频) — 200米+

每层独立: 音量/混响/EQ/延迟
```

### 5.3 音频后期处理

```
信号链:
源音效 →  [EQ]  →  [压缩]  →  [混响]  →  [延迟]  →  [限制器]  →  输出

EQ (均衡器):
├── 低频(20-200Hz): 轰隆/厚重
├── 中频(200-5kHz): 清晰度/存在感
├── 高频(5k-20kHz): 空气感/细节

混响参数:
├── 房间大小(Room Size): 小→大
├── 早期反射(Early Reflections)
├── 衰减时间(RT60): 0.5s(录音棚)~5s(大教堂)
└── 预延迟(Pre-Delay): 直达声→首次反射间隔
```

### 5.4 杜比全景声 (Dolby Atmos)

```
传统环绕: 5.1/7.1声道(水平面)
全景声: 基于对象 + 声道混合

声音对象: 3D空间坐标 + 大小 + 移动
渲染器: 根据实际音箱布局(5.1.2/7.1.4/...128声道)实时计算

游戏应用: 
- 上方声道:.2/.4顶置 = 雨水/直升机/楼上脚步声
- 对象化: 100+独立声音对象自由移动
```
