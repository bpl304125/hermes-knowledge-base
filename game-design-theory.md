# 游戏设计理论与方法论

---

## 一、MDA 框架 (Mechanics-Dynamics-Aesthetics)

游戏设计核心分析框架，由西北大学 Hunicke/LeBlanc/Zubek 提出：

```
设计师视角: Mechanics → Dynamics → Aesthetics
玩家视角:   Aesthetics → Dynamics → Mechanics
```

### 1.1 机制 (Mechanics)
游戏规则层面：生命值、攻击力、移动速度、掉落概率、经济数值

### 1.2 动态 (Dynamics)
机制交互产生的行为：资源竞争、团队配合、Min-Max优化、速通策略、Meta演变

### 1.3 美学 (Aesthetics)
玩家的情感体验：

| 美学类型 | 描述 | 代表游戏 |
|---------|------|---------|
| **Sensation** (感官) | 视听愉悦 | 音游、画面党游戏 |
| **Fantasy** (幻想) | 成为另一个人 | RPG、扮演类 |
| **Narrative** (叙事) | 引人入胜的故事 | 视觉小说、RPG |
| **Challenge** (挑战) | 克服困难 | 魂系、平台跳跃 |
| **Fellowship** (社交) | 与他人互动 | 多人合作、MMO |
| **Discovery** (探索) | 发现新事物 | 开放世界、Metroidvania |
| **Expression** (表达) | 创造/自定义 | 模拟人生、Minecraft |
| **Submission** (消遣) | 打发时间 | 三消、挂机游戏 |

## 二、心流理论 (Flow Theory)

### 2.1 Csikszentmihalyi 心流模型

```
        高难度
          │
焦虑 ←────┼────→ 无聊
          │
        低难度

心流通道: 难度 略高于 技能 → 进入心流状态
```

### 2.2 游戏心流设计

```
动态难度调节(DDA):
├── 玩家死亡→略微降低敌人AI/增加掉落
├── 玩家连胜→略微提升敌人强度
├── 自适应: 不告诉玩家，暗中调节
└── 范围: 调整在10-20%幅度内(否则被感知)

关卡节奏:
紧张(战斗) → 放松(探索/剧情) → 紧张 → 放松 → ...
```

## 三、玩家心理学

### 3.1 Bartle 玩家分类

| 类型 | 动机 | 喜欢 | 设计要素 |
|------|------|------|---------|
| **杀手(Killer)** | 支配他人 | PvP/排行榜 | 竞技系统 |
| **成就者(Achiever)** | 完成目标 | 成就/收集/等级 | 进度系统 |
| **社交者(Socializer)** | 与人互动 | 公会/聊天/交易 | 社交功能 |
| **探索者(Explorer)** | 发现秘密 | 彩蛋/隐藏/地图 | 隐藏内容 |

### 3.2 行为经济学在游戏中的应用

| 效应 | 原理 | 游戏应用 |
|------|------|---------|
| **锚定效应** | 先看到的价格影响判断 | 先显示原价→再显示折扣 |
| **损失厌恶** | 失去比获得更痛苦 | 限时活动/签到中断惩罚 |
| **峰终定律** | 记忆峰值+结尾 | BOSS战高潮+精彩结局 |
| **可变奖励** | 随机奖励最上瘾 | 抽卡/开箱/随机掉落 |
| **沉没成本** | 已投入的不愿放弃 | 战令(Battle Pass)购买后持续 |
| **社会证明** | 跟随他人行为 | "9999人已购买"/排行榜 |

### 3.3 斯金纳箱与道德边界

**避免操纵性设计**：
- ✅ 战令：玩家选择购买，回报明确
- ✅ 皮肤：纯外观，不影响平衡
- ❌ 无限抽卡无保底 → 赌博
- ❌ 刻意制造社交压力 → PUA
- ❌ 儿童定向消费诱导 → 违法

## 四、关卡设计

### 4.1 关卡设计流程

```
1. 确定体验目标(教新机制? 测试已学?)
2. 纸上设计(平面图+敌人/道具位置)
3. 白盒搭建(引擎中用灰色几何体)
4. 可玩性测试(内部试玩→迭代)
5. 美术抛光(替换白盒为最终美术)
6. 外部测试(获取真实玩家反馈)
```

### 4.2 关卡引导技巧

| 技巧 | 方法 | 例子 |
|------|------|------|
| **灯光引导** | 照亮目标方向 | Portal的光源指引 |
| **颜色对比** | 可交互物品鲜明颜色 | 镜之边缘红色引导 |
| **引导线** | 环境线条指向目标 | 道路/栏杆/电线 |
| **NPC/敌人朝向** | 朝向→吸引注意力 | 敌人巡逻方向 |
| **硬币/收集品路径** | 排列成引导线 | 马里奥金币轨迹 |
| **消极空间** | 封闭非目标区域 | 单向门/倒塌通道 |

### 4.3 难度曲线设计

```
教程关: 只引入1个机制，无惩罚
第1关: 引入第2个机制，与机制1简单组合
第2关: 两个机制复杂组合
第3关: 引入第3个机制
...
BOSS关: 测试所有已学机制，高压力
```

### 4.4 常见关卡类型

| 类型 | 特点 | 代表 |
|------|------|------|
| **线性** | A→B，顺序推进 | 横版动作 |
| **分支** | 多条路径可选 | RPG地图 |
| **开放** | 自由探索 | 开放世界 |
| **竞技场** | 封闭空间战斗 | 清版动作 |
| **Metroidvania** | 获得能力解锁新区域 | 空洞骑士 |
| **随机生成** | 每次不同 | Roguelike |

## 五、数值设计

### 5.1 核心循环

```
战斗 → 获得资源 → 强化角色 → 更强战斗 → (循环)

每个循环必须:
├── 反馈明确(数字变化可视化)
├── 时间合理(5-15分钟一个小循环)
└── 成长可感(战力提升后的碾压体验)
```

### 5.2 数值平衡方法

```
Excel模拟法:
├── 列出所有变量(攻击/防御/血量/速度)
├── 计算DPS = 攻击力 × 攻击速度
├── 计算TTK(Time To Kill) = 血量 / DPS
├── 调整使TTK在目标范围内(3-5秒)
└── 考虑边际效应和反超情况

边际效应:
属性   伤害   每点收益
100→200  +100%  100%  ← 初期高效
200→300  +50%   50%   ← 收益递减
300→400  +33%   33%

对策:
├── 线性成长(简单，容易平衡)
├── 边际递减(自然防通胀)
└── 关键拐点(特定数值触发质变)
```

### 5.3 经济系统设计

```
游戏经济回路:
             ┌──→ 任务奖励 ──┐
             │               │
玩家劳动 → 金币/经验 → 商店消费 → 更强战力
    ↑                               │
    └──────── 需要更强 ──────────────┘

关键参数:
- 金币产出率: 每小时期望获取量
- 金币消耗: 物价定位(消耗品<永久道具<奢侈装饰品)
- 通胀控制: 限制无限产出，定期出金回收
```

## 六、游戏类型设计模式

### 6.1 Roguelike/Roguelite

```
核心:
├── 永久死亡(Permadeath) 或 有限复活
├── 程序化生成关卡(每次不同)
├── 回合制或即时制
├── 资源管理(饥饿/道具/装备)
└── RPG成长(升级/技能/装备)

Roguelite(轻度): 死亡保留部分进度(解锁/货币)
Roguelike(硬核): 死亡全部重置
```

### 6.2 类魂 (Souls-like)

```
核心:
├── 高难度: 敌人3-5击可杀玩家
├── 死亡惩罚: 丢失经验/货币(可拾回)
├── 检查点: 篝火/存档点，敌人重生
├── 耐力管理: 攻击/格挡/闪避消耗共用耐力
├── 战斗节奏: 观察→闪避→反击→观察循环
└── 环境叙事: 物品描述+场景+碎片化剧情
```

### 6.3 生存建造

```
核心循环:
探索 → 采集 → 建造 → 防御 → 探索(更深/更远)

技术树:
基础工具(石斧) → 熔炉 → 金属工具 → 机械 → 自动化
```

## 七、游戏测试

### 7.1 测试类型

| 类型 | 问题 | 阶段 |
|------|------|------|
| **纸面测试** | 规则是否有漏洞? | 设计阶段 |
| **内部QA** | Bug/崩溃/卡关 | 开发全程 |
| **可玩性测试** | 好玩吗? | 每2-4周 |
| **盲测** | 没有指导能玩懂吗? | 发布前 |
| **A/B测试** | 哪个版本更好? | 线上运营 |
| **压力测试** | 1000人同时在线? | 发布前 |

### 7.2 可玩性测试流程

```
1. 不指导玩家，只观察
2. 记录: 哪里卡住/困惑/死亡/重复操作
3. 问题分级: 阻塞性(必修)/重要/建议
4. 迭代修改 → 再次测试
```

## 八、独立游戏商业

### 8.1 平台策略

| 平台 | 分成 | 曝光 | 适合 |
|------|------|------|------|
| **Steam** | 30%(<1000万→25%) | 算法推荐+愿望单 | PC首选 |
| **itch.io** | 开发者自定(0-30%) | 社区驱动 | 试水/独立 |
| **Epic** | 12% | 需邀请 | 独占策略 |
| **抖音小游戏** | 字节分成 | 短视频裂变 | 休闲/超休闲 |
| **微信小游戏** | 腾讯分成 | 社交裂变 | 休闲/IAA |
| **App Store** | 30%(小企业15%) | 编辑推荐 | 精品手游 |
| **Google Play** | 15%(首百万)+30% | 算法推荐 | 全球手游 |

```
关键指标:
- 愿望单: 发布前积累5000+
- 发布首周销量 ≈ 愿望单 × 0.2~0.5
- 需要愿望单5000-10000才有机会进入"热门即将推出"

获取愿望单:
├── Steam Next Fest: 发布Demo(最重要!)
├── 社交媒体: Twitter/TikTok/Reddit/B站
├── 媒体: 发送Key给YouTuber/主播
├── 社区: Discord社区预建
└── 商店页: 高质量预告片+截图+描述
```
