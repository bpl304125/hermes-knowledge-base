# 游戏经济系统设计 & 程序化建筑生成

## 一、城市模拟经济平衡

### 供需曲线
```
人口增长 → 住房需求↑ → 房价↑ → 需要更多住宅区
         → 商品需求↑ → 商铺收入↑ → 需要更多商业区
         → 就业需求↑ → 工厂产出↑ → 需要更多工业区

税收 = (人口×税率) + (贸易×关税率) - (基础设施维护费)
净收入 > 0 → 持续扩张
净收入 < 0 → 破产!
```

### 满意度公式
```
满意度 = 基础值(50)
  + 公园/娱乐×1.5
  + 教育覆盖×2
  - 污染(工厂×3 + 工业密度)
  - 拥堵(道路密度过高)
  - 税率(高税收扣分)
  + 医疗/安全覆盖
```

## 二、程序化建筑生成算法

### WFC(波函数坍缩)建筑布局
```
1. 定义相邻规则: 住宅旁不能有工厂
2. 从熵最小(约束最多)地块开始坍缩
3. 传播约束到邻居
4. 循环直到全部确定
```

### 建筑变体系统
```
同一住宅类型 → 随机选择:
- 屋顶颜色(红/灰/蓝)
- 楼层数(1-3随机)
- 阳台/窗户样式
- 花园/围栏装饰
→ 每个建筑视觉不同但功能相同
```

## 三、政策系统设计

| 政策 | 效果 | 成本 |
|------|------|------|
| 减税令 | 满意度+15, 税收-20% | 0 |
| 环保令 | 工厂污染-50%, 维护+30% | 500/月 |
| 教育补贴 | 学校容量×2 | 300/月 |
| 工业振兴 | 工厂产出+30% | 1000/月 |
| 宵禁 | 安全+20, 商业收入-10% | 200/月 |
| 旅游推广 | 贸易+50%, 地标收入×2 | 800/月 |
